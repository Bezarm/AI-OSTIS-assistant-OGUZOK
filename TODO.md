
## Проблематика
- аудитории
	- студенты без опыта в готовке, желающие питаться полезнее и бюджетно
	- люди, ищущие новые решения, идеи; подобные блюда
	- люди, которые не умеют готовить, но хотят научиться
- точность рецептов, в отличии от предложенных LLM
- возможность предложения малоизвестных блюд
- отсутствие анализа изображений и гибкой маштабируемости базы заний у аналогов
## План работы с пользователем
- `/start` - возможности бота
- опрос по аллергиям и другим постоянным исключениям и предпочтительным ингредиентам
- `/fastcook` или отправлены картинки
	- +ручной выбор ингредиентов
		- миниапп и/или категории?
		- текстовое добавление с ostis?
	- предложения по блюдам с ostis
		- выбор: нужны ли предложения по доп. продуктам
		- *если да, предложить ближайшие варианты покупки*
			- подтянуть цены, спарсить
				- подбор по бюджету...
			- API с заказами (например, яндекс еда)
		- *система рекомендаций блюд*
	- описание блюда (название, калории, *история*, ингредиенты, время приготовления, сложность)
- текстовый запрос напрямую без команды
	- предложения идей блюд по запросу (*с рекомендациями блюд*)
		- рецепты похожих, близких блюд
	- сводить к выбору ингредиентов
- начало интерактивной готовки после выбора рецепта
	- бот присылает пошагово, например `[Добавьте картошку]`
	- внизу находятся кнопки `[Назад]` и `[Далее]`
	- интерграция с таймерами
- *сравнения цен в ближайших магазинах*
